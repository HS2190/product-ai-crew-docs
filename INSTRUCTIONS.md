# Product AI Crew — 에이전트 구조 요약

> 각 에이전트의 역할과 인풋/아웃풋 흐름을 정리한 문서입니다.

---

## 오케스트레이터 (Orchestrator)

사용자의 요청을 받아 모드를 판단하고, 에이전트를 순서대로 호출해 전체 파이프라인을 관리합니다.
각 에이전트의 아웃풋을 다음 에이전트의 인풋으로 전달하며, 세션 상태를 `session.md`로 관리합니다.

| 실행 모드 | 에이전트 호출 순서 | 사용 상황 |
| --- | --- | --- |
| FULL | PM → 기획자 → 디자이너 → UX라이터 | 신규 서비스/기능 전체 |
| PLAN | PM → 기획자 | PRD + 화면 기획까지 |
| DESIGN | 기획자 → 디자이너 | 기획안이 있고 디자인만 필요 |
| WRITE | UX라이터 단독 | 문구 작업만 |
| CUSTOM | 사용자 정의 | 에이전트 직접 조합 |

---

## PM (Product Manager)

"무엇을 왜 만들 것인가"를 정의하고 제품 전략과 우선순위를 결정합니다.

| 구분 | 항목 |
| --- | --- |
| 인풋 | 서비스명, 기능명, 비즈니스 목표, 타겟 사용자 |
| 아웃풋 | PRD, 기능 목록, 우선순위, 브랜드 방향성 |
| 다음 에이전트 | 기획자 |

산출물: PRD · 기능 우선순위 정의서 (Must Have / Should Have / Nice to Have) · 제품 로드맵

---

## 기획자 (Product Planner)

"어떻게 만들 것인가"를 정의하고 화면 기획과 기능 명세서를 작성합니다.

| 구분 | 항목 |
| --- | --- |
| 인풋 | PRD, 기능 목록, 우선순위, 플랫폼, 브랜드 방향성 |
| 아웃풋 | 화면 기획안, 기능 명세서, User Flow, 와이어프레임 |
| 다음 에이전트 | 디자이너 |

산출물: 화면 기획안 (Figma) · 기능 명세서 (트리거/동작/예외 처리 포함) · User Flow (Figma / FigJam)

---

## 디자이너 (Product Designer)

### 역할 — 화면을 시각적으로 설계하고 컴포넌트 스펙을 정의합니다.

| 구분 | 항목 |
| --- | --- |
| 인풋 | 화면 기획안, 기능 명세서, 플랫폼, 브랜드 방향성 |
| 아웃풋 | 디자인 스펙, 컴포넌트 스펙, Figma URL |
| 다음 에이전트 | UX 라이터 |

산출물: 디자인 스펙 문서 · 컴포넌트 스펙 (default / hover / active / disabled / error) · Figma 파일

---

## UX 라이터 (UX Writer)

### 역할 — UI 문구를 설계하고 보이스 톤을 일관되게 관리합니다. 파이프라인의 최종 아웃풋을 담당합니다.

| 구분 | 항목 |
| --- | --- |
| 인풋 | 화면 기획안, Figma URL, 브랜드 방향성, 보이스 가이드 |
| 아웃풋 | UX 라이팅 가이드, 화면별 문구 시트 |
| 다음 에이전트 | 없음 (파이프라인 종료) |

산출물: UX 라이팅 가이드 (보이스 퍼스널리티 · 라이팅 원칙 · 금지 표현 목록) · 화면별 문구 시트 (원문 / 수정안 / 수정 이유)

---

## 인풋/아웃풋 전체 흐름

```text
[사용자 요청]
      ↓
[오케스트레이터] — 모드 판단 · 세션 확인 · 에이전트 순서 결정
      ↓
[PM]        인풋: 서비스·기능·목표·사용자
            아웃풋: PRD · 기능 목록 · 우선순위 · 브랜드 방향성
      ↓
[기획자]    인풋: PRD · 기능 목록 · 플랫폼 · 브랜드 방향성
            아웃풋: 화면 기획안 · 기능 명세서 · User Flow
      ↓
[디자이너]  인풋: 화면 기획안 · 기능 명세서 · 플랫폼 · 브랜드 방향성
            아웃풋: 디자인 스펙 · 컴포넌트 스펙 · Figma URL
      ↓
[UX라이터]  인풋: 화면 기획안 · Figma URL · 브랜드 방향성 · 보이스 가이드
            아웃풋: UX 라이팅 가이드 · 화면별 문구 시트
      ↓
[작업 완료 보고]
```
